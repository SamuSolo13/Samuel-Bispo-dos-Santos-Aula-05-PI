## Criação do repositório
O que é: é o processo de criar um projeto dentro do GitHub, onde ficarão armazenados todos os arquivos, histórico de alterações e configurações.

Para que serve: serve como base do projeto, permitindo salvar versões do código ao longo do tempo, além de facilitar o trabalho em equipe, já que várias pessoas podem acessar e contribuir no mesmo lugar.

## Clone do repositório
O que é: é a ação de copiar um repositório que está no GitHub para o seu computador.

Para que serve: permite que você trabalhe localmente no projeto, editando arquivos, testando e fazendo alterações antes de enviar para o repositório online.

## Criação de branch
O que é: uma branch é uma “linha paralela” de desenvolvimento dentro do mesmo projeto.

Para que serve: possibilita desenvolver novas funcionalidades ou corrigir erros sem interferir diretamente na versão principal do projeto, evitando problemas no código principal.

## Registro de commits
O que é: o commit é o registro das alterações feitas nos arquivos do projeto.

Para que serve: cada commit cria um ponto no histórico, permitindo acompanhar o que foi modificado, quando foi feito e por quem. Isso ajuda a organizar o desenvolvimento e até voltar para versões anteriores, se necessário.

## Envio das alterações (push)
O que é: é o envio dos commits feitos no seu computador para o repositório no GitHub.

Para que serve: garante que suas alterações fiquem salvas online e disponíveis para outras pessoas da equipe.

## Atualização do repositório local (pull)
O que é: é o processo de baixar e integrar no seu computador as alterações feitas no repositório remoto.

Para que serve: mantém seu projeto atualizado, evitando conflitos com mudanças feitas por outras pessoas.

## Criação de Issues
O que é: são registros usados para descrever tarefas, problemas, bugs ou melhorias no projeto.

Para que serve: ajudam na organização do trabalho, permitindo que a equipe saiba o que precisa ser feito e acompanhe o andamento das atividades.

## Organização com Milestone
O que é: é um recurso que agrupa várias issues dentro de um objetivo maior, como uma versão do projeto.

Para que serve: facilita o planejamento e acompanhamento de metas, mostrando o progresso de uma etapa específica.

## Gerenciamento com Project
O que é: é uma ferramenta visual do GitHub, semelhante a um quadro kanban.

Para que serve: organiza as tarefas em colunas (por exemplo: “a fazer”, “em andamento”, “concluído”), ajudando a visualizar o fluxo de trabalho e o progresso do projeto.

## Abertura de Pull Request
O que é: é uma solicitação para que alterações feitas em uma branch sejam analisadas e incorporadas em outra branch (geralmente a principal).

Para que serve: permite revisar o código antes de aceitar as mudanças, garantindo maior qualidade e evitando erros no projeto.

## Realização de Merge
O que é: é a ação de juntar as alterações de uma branch com outra.

Para que serve: serve para integrar definitivamente as mudanças ao projeto principal após aprovação, mantendo o código atualizado.
